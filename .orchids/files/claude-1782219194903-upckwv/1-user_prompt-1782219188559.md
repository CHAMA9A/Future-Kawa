On ajoute maintenant Swagger / OpenAPI au projet FutureKawa.

Contexte :
Le projet contient :

* backend-country/brazil-service
* backend-country/ecuador-service
* backend-country/colombia-service
* backend-central

Objectif :
Ajouter une documentation Swagger propre pour les APIs NestJS.

Important :

* Ne pas modifier la logique métier.
* Ne pas modifier les routes existantes.
* Ne pas casser les tests existants.
* Ne pas modifier le frontend.
* Ne pas ajouter d’authentification.
* Ne pas ajouter de nouveau backend.
* Ajouter uniquement Swagger/OpenAPI et la documentation associée.

Travail attendu :

1. Installer Swagger dans les backends NestJS

Dans chaque backend concerné :

* backend-central
* brazil-service
* ecuador-service
* colombia-service

Installer si nécessaire :

npm install @nestjs/swagger swagger-ui-express

2. Configurer Swagger dans `main.ts`

Pour chaque backend, ajouter une configuration Swagger propre.

Exemple attendu :

```ts
import { SwaggerModule, DocumentBuilder } from '@nestjs/swagger';

const config = new DocumentBuilder()
  .setTitle('FutureKawa API')
  .setDescription('API documentation for FutureKawa services')
  .setVersion('1.0')
  .build();

const document = SwaggerModule.createDocument(app, config);
SwaggerModule.setup('api/docs', app, document);
```

Adapter le titre selon le service :

* FutureKawa Brazil Service API
* FutureKawa Ecuador Service API
* FutureKawa Colombia Service API
* FutureKawa Central API

3. URLs Swagger attendues

Après lancement Docker :

* Brazil service Swagger :
  http://localhost:3001/api/docs

* Ecuador service Swagger :
  http://localhost:3011/api/docs

* Colombia service Swagger :
  http://localhost:3012/api/docs

* Backend central Swagger :
  http://localhost:3002/api/docs

4. Ajouter des décorateurs Swagger dans les controllers

Ajouter au minimum :

* `@ApiTags()`
* `@ApiOperation()`
* `@ApiResponse()`

Sur les controllers principaux :

Backends pays :

* LotsController
* MeasurementsController
* AlertsController

Backend central :

* CentralController

Exemples :

* `@ApiTags('Lots')`
* `@ApiOperation({ summary: 'Get all coffee lots' })`
* `@ApiResponse({ status: 200, description: 'List of coffee lots returned successfully' })`

5. Ajouter des DTO documentés si possible

Si des DTO existent, ajouter :

* `@ApiProperty()`

Exemples :

* CreateLotDto
* CreateMeasurementDto

Documenter les champs :

* lotCode
* country
* warehouseName
* storageDate
* temperature
* humidity

Créer ou mettre à jour :

docs/api-documentation.md

Ce fichier doit expliquer :

* objectif de Swagger
* URLs Swagger par service
* principales routes API
* comment tester une route depuis Swagger
* comment Swagger aide à documenter le projet MSPR

7. README

Mettre à jour README.md avec une section :

Inclure :
