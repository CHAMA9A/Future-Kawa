Je travaille sur une MSPR appelée FutureKawa.

Le cahier des charges demande une solution applicative pour suivre les stocks de café vert et les conditions de stockage dans plusieurs pays : Brésil, Équateur et Colombie.

Je ne veux pas utiliser Java Spring Boot. Je veux utiliser cette stack :

* Backend pays : NestJS
* Backend central : NestJS
* Frontend : React + TypeScript
* Base de données : PostgreSQL
* ORM : Prisma
* MQTT : Mosquitto
* Docker : Docker Compose
* Tests : Jest
* CI/CD : Jenkins

Objectif du projet :

Créer une architecture distribuée pays / siège.

Pour le prototype, on développe complètement un backend pays exemple : Brésil.

Le backend Brésil doit contenir :

1. Gestion des lots de café :

   * créer un lot
   * afficher tous les lots
   * afficher les lots triés par date de stockage selon la logique FIFO
   * chaque lot doit avoir :

     * id
     * lotCode
     * country
     * warehouseName
     * storageDate
     * status : CONFORME, EN_ALERTE ou PERIME

2. Gestion des mesures IoT :

   * recevoir température et humidité
   * stocker les mesures dans PostgreSQL
   * chaque mesure doit avoir :

     * id
     * warehouseName
     * temperature
     * humidity
     * measuredAt

3. Gestion des alertes :

   * créer une alerte si la température ou l’humidité dépasse les seuils du Brésil
   * seuils Brésil :

     * température idéale : 29°C
     * tolérance température : ±3°C
     * humidité idéale : 55%
     * tolérance humidité : ±2%
   * créer une alerte si un lot dépasse 365 jours de stockage
   * simuler l’envoi d’un email dans les logs

4. MQTT :

   * utiliser Mosquitto
   * écouter le topic :
     futurekawa/brazil/warehouse-1/measurements
   * payload attendu :
     {
     "warehouseName": "Warehouse Brazil 1",
     "temperature": 34,
     "humidity": 60
     }
   * quand un message MQTT arrive, le backend doit enregistrer la mesure et créer une alerte si nécessaire.

5. Backend central :

   * créer un backend NestJS séparé
   * il doit interroger le backend Brésil via API REST
   * routes attendues :

     * GET /api/central/brazil/lots
     * GET /api/central/brazil/measurements
     * GET /api/central/brazil/alerts

6. Frontend React :

   * page Dashboard
   * page Lots
   * page Measurements avec courbes température/humidité
   * page Alerts
   * le frontend doit interroger le backend central

7. Docker :

   * créer un docker-compose.yml à la racine
   * services :

     * postgres-brazil
     * mosquitto-brazil
     * backend-brazil
     * backend-central
     * frontend

8. Tests :

   * tests unitaires NestJS avec Jest
   * tester :

     * création d’un lot
     * tri FIFO
     * création d’une mesure
     * détection alerte température
     * détection alerte humidité
     * détection lot périmé

9. Jenkins :

   * créer un Jenkinsfile simple
   * étapes :

     * install dependencies
     * run tests
     * build backend
     * build frontend
     * build Docker images

10. Documentation :

* README.md
* docs/architecture/architecture-globale.md
