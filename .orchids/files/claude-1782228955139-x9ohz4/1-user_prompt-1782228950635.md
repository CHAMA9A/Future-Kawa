On veut maintenant connecter le prototype Arduino UNO + DHT11 directement aux données de l’application FutureKawa.

Contexte :
Le prototype Arduino UNO + DHT11 fonctionne.
Le Serial Monitor affiche des valeurs réelles comme :

```txt
Humidity: 42.00%  Temperature: 27.50°C
```

Objectif :
Créer une passerelle locale Serial-to-MQTT pour lire les données envoyées par l’Arduino via USB/port série, puis publier automatiquement ces mesures dans Mosquitto MQTT afin que l’application FutureKawa les enregistre dans le backend pays, PostgreSQL, backend-central et frontend.

Important :

* Ne pas modifier le code Arduino existant.
* Ne pas modifier les backends.
* Ne pas modifier le frontend.
* Ne pas modifier Docker Compose.
* Ne pas dire que l’Arduino UNO publie directement en MQTT.
* L’Arduino UNO envoie les données en Serial USB.
* Le script PC lit le Serial et publie vers MQTT.
* Le script doit être simple, documenté, et utilisable sur Windows PowerShell.

Créer un script Node.js `serial-to-mqtt.js`.

Fonctionnement attendu :

1. Lire le port série Arduino.
2. Parser les lignes du type :
   `Humidity: 42.00%  Temperature: 27.50°C`
3. Extraire :

   * humidity = 42.00
   * temperature = 27.50
4. Construire le payload JSON :

   ```json
   {
     "warehouseName": "Warehouse Colombia 1",
     "temperature": 27.5,
     "humidity": 42
   }
   ```
5. Publier ce payload vers MQTT.

Dépendances Node.js :

* serialport
* @serialport/parser-readline
* mqtt
* dotenv

Configuration via `.env` :

```env
SERIAL_PORT=COM4
SERIAL_BAUD_RATE=9600
MQTT_URL=mqtt://localhost:1885
MQTT_TOPIC=futurekawa/colombia/warehouse-1/measurements
WAREHOUSE_NAME=Warehouse Colombia 1
```

Important :

* Pour Colombia, depuis le PC Windows vers Docker, le port externe Mosquitto est 1885.
* Donc l’URL MQTT locale doit être :
  `mqtt://localhost:1885`

Le script doit afficher dans la console :

* connexion au port série
* connexion MQTT
* chaque ligne reçue de l’Arduino
* chaque payload publié
* les erreurs éventuelles

Exemple de logs attendus :

```txt
Serial connected on COM4 at 9600 baud
MQTT connected to mqtt://localhost:1885
Arduino line: Humidity: 42.00%  Temperature: 27.50°C
Published to futurekawa/colombia/warehouse-1/measurements: {"warehouseName":"Warehouse Colombia 1","temperature":27.5,"humidity":42}
```

Créer `package.json` avec scripts :

```json
{
  "scripts": {
    "start": "node serial-to-mqtt.js"
  }
}
```

Expliquer :
La passerelle Serial-to-MQTT permet d’intégrer les mesures physiques du capteur DHT11 directement dans la chaîne applicative FutureKawa sans modifier le backend. Le PC lit les données série envoyées par l’Arduino UNO puis les publie dans Mosquitto au format JSON attendu par les services pays.
