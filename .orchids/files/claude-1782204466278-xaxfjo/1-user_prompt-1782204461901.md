On passe à l’étape 8 du projet FutureKawa : tests techniques et fonctionnels.

Contexte :
Le projet contient maintenant :

* backend-country/brazil-service
* backend-country/ecuador-service
* backend-country/colombia-service
* backend-central
* frontend React
* Docker Compose avec PostgreSQL, Mosquitto, backends et frontend

Objectif :
Ajouter une vraie base de tests et une documentation des scénarios de test pour prouver que le projet fonctionne.

Important :
Ne pas modifier la logique métier existante.
Ne pas casser Docker.
Ne pas ajouter Jenkins maintenant.
Ne pas ajouter de nouveau backend.
Ne pas ajouter d’authentification.
Ne pas intégrer le capteur réel maintenant.
On reste uniquement sur les tests.

Travail attendu :

1. Tests backend pays

Pour chaque service pays :

* brazil-service
* ecuador-service
* colombia-service

Ajouter ou compléter les tests Jest pour vérifier :

* création d’un lot
* récupération des lots
* récupération FIFO des lots
* création ou récupération des mesures
* récupération des alertes

Tester les services principaux si possible :

* LotsService
* MeasurementsService
* AlertsService

2. Tests backend central

Dans backend-central, ajouter des tests pour vérifier :

* récupération de la liste des pays
* appel des lots Brazil
* appel des lots Ecuador
* appel des lots Colombia
* gestion d’erreur si un service pays ne répond pas

Utiliser des mocks HTTP si nécessaire, pour ne pas dépendre des vrais containers dans les tests unitaires.

3. Tests frontend

Dans frontend, ajouter au minimum des tests simples pour :

* le composant CountrySelector
* l’affichage des DashboardCards
* l’affichage des tableaux avec données vides
* l’affichage du message d’erreur si l’API ne répond pas

Utiliser Vitest ou React Testing Library si compatible avec Vite.

4. Documentation des tests

Créer un fichier :

docs/tests.md

Ce fichier doit expliquer les scénarios de test manuels :

A. Test Docker Compose
Commande :
docker compose up -d --build

Résultat attendu :
Tous les containers sont Up :

* frontend
* backend-central
* brazil-service
* ecuador-service
* colombia-service
* postgres-brazil
* postgres-ecuador
* postgres-colombia
* mosquitto-brazil
* mosquitto-ecuador
* mosquitto-colombia

B. Test création de lots

Inclure les commandes PowerShell pour créer un lot dans chaque pays :

Brazil :
POST http://localhost:3001/api/lots

Ecuador :
POST http://localhost:3011/api/lots

Colombia :
POST http://localhost:3012/api/lots

C. Test FIFO

Inclure les routes :
GET http://localhost:3001/api/lots/fifo
GET http://localhost:3011/api/lots/fifo
GET http://localhost:3012/api/lots/fifo

Résultat attendu :
Les lots les plus anciens doivent apparaître en premier.

D. Test MQTT

Inclure les commandes Mosquitto pour publier des mesures :

Brazil topic :
futurekawa/brazil/warehouse-1/measurements

Ecuador topic :
futurekawa/ecuador/warehouse-1/measurements

Colombia topic :
futurekawa/colombia/warehouse-1/measurements

Format JSON attendu :
{
"warehouseName": "Warehouse Colombia 1",
"temperature": 30,
"humidity": 85
}

E. Test alertes

Expliquer les seuils :
Brazil :

* température acceptable : 26°C - 32°C
* humidité acceptable : 53% - 57%

Ecuador :

* température acceptable : 28°C - 34°C
* humidité acceptable : 58% - 62%

Colombia :

* température acceptable : 23°C - 29°C
* humidité acceptable : 78% - 82%

Résultat attendu :
Si une mesure dépasse les seuils, une alerte est créée.

F. Test backend central

Inclure les routes :
GET http://localhost:3002/api/central/countries
GET http://localhost:3002/api/central/brazil/lots
GET http://localhost:3002/api/central/ecuador/lots
GET http://localhost:3002/api/central/colombia/lots
GET http://localhost:3002/api/central/brazil/alerts
GET http://localhost:3002/api/central/ecuador/alerts
GET http://localhost:3002/api/central/colombia/alerts

G. Test frontend

URL :
http://localhost:5173

Vérifier :

* dashboard visible
* sélection Brazil / Ecuador / Colombia
* cartes statistiques affichées
* lots affichés
* mesures affichées
* alertes affichées
* aucun message “Unable to load data from central backend” si le backend central fonctionne
