Le problème OpenSSL est corrigé, mais il reste une erreur Prisma sur les 3 backends pays.

Erreur actuelle :
PrismaClientInitializationError: Prisma Client could not locate the Query Engine for runtime "debian-openssl-3.0.x".

Explication :
Prisma Client a été généré pour "linux-musl", mais maintenant les Dockerfiles utilisent node:22-slim, donc Debian. Il faut donc générer Prisma Client pour "debian-openssl-3.0.x".

Merci de corriger les 3 fichiers schema.prisma :

backend-country/brazil-service/prisma/schema.prisma
backend-country/ecuador-service/prisma/schema.prisma
backend-country/colombia-service/prisma/schema.prisma

Dans chaque fichier, mettre :

generator client {
provider      = "prisma-client-js"
binaryTargets = ["native", "debian-openssl-3.0.x"]
}

Ensuite vérifier que les Dockerfiles utilisent node:22-slim dans le build stage et le runtime stage.

Important :

* Ne pas utiliser node:22-alpine.
* Ne pas copier node_modules local dans l’image Docker.
* Ajouter node_modules dans .dockerignore si ce n’est pas déjà fait.
* Faire npx prisma generate dans le Dockerfile après COPY prisma ./prisma.
* Garder CMD ["sh", "-c", "npx prisma migrate deploy && node dist/main.js"].

Après correction, lancer :

docker compose build --no-cache brazil-service ecuador-service colombia-service

docker compose up -d --force-recreate brazil-service ecuador-service colombia-service

docker ps
