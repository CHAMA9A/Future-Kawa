Dans `backend-central/src/main.ts`, merci de vérifier et corriger la configuration CORS.

Contexte :

* Le backend central tourne correctement sur http://localhost:3002
* Les logs du backend central sont OK
* Le frontend tourne avec Docker sur http://localhost:5173
* Mais le frontend affiche encore : "Unable to load data from central backend"

Objectif :
Autoriser le frontend Vite/Docker à appeler le backend central.

Merci de modifier `backend-central/src/main.ts` pour avoir une configuration CORS propre, par exemple :

```ts
app.enableCors({
  origin: [
    'http://localhost:5173',
    'http://127.0.0.1:5173',
  ],
  methods: ['GET', 'POST', 'PUT', 'PATCH', 'DELETE', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization'],
});
```

Ou temporairement, pour vérifier que le problème vient bien de CORS :

```ts
app.enableCors();
```

Important :

* Ne pas modifier la logique métier du backend central.
* Ne pas modifier les backends pays.
* Ne pas créer de nouveau backend.
* Modifier uniquement la configuration CORS dans `backend-central/src/main.ts` si nécessaire.

Après modification, il faudra rebuild et relancer :

```powershell
docker compose build --no-cache backend-central frontend
docker compose up -d backend-central frontend
```
