On passe à l’étape 6 du projet FutureKawa : création du frontend React.

Important :
Le backend multi-pays est déjà terminé et mergé dans main.

Architecture existante :

* backend-country/brazil-service : http://localhost:3001
* backend-country/ecuador-service : http://localhost:3011
* backend-country/colombia-service : http://localhost:3012
* backend-central : http://localhost:3002
* docker-compose.yml contient déjà les services backend, PostgreSQL et Mosquitto.

Objectif :
Créer un frontend React + TypeScript avec Vite dans le dossier :

frontend/

Le frontend doit appeler uniquement le backend central :
http://localhost:3002

Ne pas appeler directement les backends pays.
Ne pas créer de backend.
Ne pas utiliser Supabase.
Ne pas ajouter d’authentification.
Ne pas ajouter Jenkins maintenant.

Routes API à consommer :

GET /api/central/countries
GET /api/central/:country/lots
GET /api/central/:country/lots/fifo
GET /api/central/:country/measurements
GET /api/central/:country/alerts

Pays disponibles :

* brazil
* ecuador
* colombia

Fonctionnalités attendues :

1. Dashboard principal
   Afficher :

* pays sélectionné
* nombre de lots
* nombre de mesures
* nombre d’alertes
* statut du service

   Permettre de choisir :

* Brazil
* Ecuador
* Colombia

* lots
* lots FIFO
* mesures
* alertes

3. LotsTable
   Colonnes :

* id
* lotCode
* country
* warehouseName
* storageDate
* status

   Colonnes :

* lotCode

* warehouseName
* status

5. MeasurementsTable
   Colonnes :

* id
* warehouseName
* temperature
* humidity
* measuredAt ou createdAt

6. AlertsTable
   Colonnes :

* id
* type
* message

* createdAt

* EXPIRED_LOT

* responsive design
