1. Fichier `futurekawa_dht11_serial.ino`

Ajouter le code Arduino suivant :

```cpp
#include <DHT.h>

#define DHTPIN 7
#define DHTTYPE DHT11

DHT dht(DHTPIN, DHTTYPE);

void setup() {
  Serial.begin(9600);
  dht.begin();
}

void loop() {
  float h = dht.readHumidity();
  float t = dht.readTemperature();

  Serial.print("Humidity: ");
  Serial.print(h);
  Serial.print("%  Temperature: ");
  Serial.print(t);
  Serial.println("°C");

  delay(500);
}
```

Ce code doit être présenté comme le code de lecture du capteur physique.
Il permet :

* d’initialiser le DHT11
* de lire l’humidité
* de lire la température
* d’afficher les valeurs dans le Serial Monitor

2. Fichier `futurekawa_dht11_mqtt_bridge_template.md`

Créer une documentation qui explique l’évolution MQTT possible.

Expliquer clairement :
Arduino UNO ne possède pas de Wi-Fi intégré. Pour envoyer les données vers MQTT, il faut une passerelle.

Solutions possibles :

* Arduino UNO + script PC Serial-to-MQTT
* Arduino UNO + module Wi-Fi ESP8266
* remplacement par ESP32 / ESP8266

Expliquer que l’option la plus simple avec le matériel actuel est :

Arduino UNO + DHT11 → USB Serial → PC → script MQTT → Mosquitto FutureKawa

Donner le payload MQTT attendu par FutureKawa :

```json
{
  "warehouseName": "Warehouse Colombia 1",
  "temperature": 30,
  "humidity": 85
}
```

Topic MQTT cible :

futurekawa/colombia/warehouse-1/measurements

Port MQTT Colombia côté Docker :

1885

Adresse du broker :
Ne pas utiliser localhost depuis une carte externe.
Utiliser l’adresse IP locale du PC Docker, par exemple :
192.168.1.25


```powershell
'{"warehouseName":"Warehouse Colombia 1","temperature":30,"humidity":85}' | docker exec -i mosquitto-colombia mosquitto_pub -h localhost -t futurekawa/colombia/warehouse-1/measurements -l
```
