On passe maintenant à l’étape 4 du projet FutureKawa.

Contexte :
Le backend Brésil existe déjà dans :

backend-country/brazil-service

Il contient déjà :

* NestJS
* Prisma
* PostgreSQL
* routes lots :

  * POST /api/lots
  * GET /api/lots
  * GET /api/lots/fifo
* routes measurements :

  * POST /api/measurements
  * GET /api/measurements
  * GET /api/measurements/warehouse/:warehouseName
* routes alerts :

  * GET /api/alerts
  * POST /api/alerts/check-expired-lots
* logique d’alerte température/humidité
* simulation email dans les logs

Objectif de l’étape 4 :
Ajouter l’intégration MQTT avec Mosquitto.

Important :
Ne pas ajouter le backend central.
Ne pas ajouter le frontend.
Ne pas ajouter Jenkins.
Ne pas ajouter le vrai module IoT matériel pour l’instant.
On fait seulement la réception MQTT côté backend Brésil.

Ce que je veux :

1. Installer la dépendance MQTT :

* npm install mqtt

2. Ajouter dans .env.example :

MQTT_BROKER_URL="mqtt://localhost:1883"
MQTT_TOPIC="futurekawa/brazil/warehouse-1/measurements"

3. Créer un module MQTT dans NestJS :

src/mqtt/
├── mqtt.module.ts
└── mqtt.service.ts

4. Le service MQTT doit :

* se connecter au broker Mosquitto avec MQTT_BROKER_URL
* s’abonner au topic MQTT_TOPIC
* écouter les messages entrants
* parser le JSON reçu
* vérifier que le payload contient :

  * warehouseName
  * temperature
  * humidity
* appeler le MeasurementsService existant pour enregistrer la mesure
* réutiliser la logique existante d’alertes température/humidité

5. Modifier MeasurementsModule si nécessaire pour exporter MeasurementsService.

6. Modifier AppModule pour importer MqttModule.

7. Ajouter des logs clairs :

[MQTT] Connexion au broker réussie
[MQTT] Abonnement au topic : futurekawa/brazil/warehouse-1/measurements
[MQTT] Message reçu : ...
[MQTT] Mesure enregistrée depuis MQTT

8. Gérer les erreurs :

* si le JSON est invalide, afficher un log d’erreur sans arrêter le backend
* si le payload est incomplet, afficher un log d’erreur
* si le broker est indisponible, afficher un log clair

9. Mettre à jour le README avec :

* variables d’environnement MQTT
* commande pour lancer Mosquitto avec Docker Compose
* commande pour tester l’envoi MQTT avec mosquitto_pub

Commande de test attendue sous PowerShell :

docker exec mosquitto-brazil mosquitto_pub -h localhost -t futurekawa/brazil/warehouse-1/measurements -m '{"warehouseName":"Warehouse Brazil 1","temperature":34,"humidity":60}'

10. Expliquer brièvement le rôle de chaque nouveau fichier.

Ne pas modifier la logique REST existante.
Ne pas casser les routes déjà fonctionnelles.
