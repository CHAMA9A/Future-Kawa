On passe maintenant à l’étape 3 du projet FutureKawa.

Le backend Brésil est déjà créé dans :

backend-country/brazil-service

Il contient déjà :

* NestJS
* Prisma
* PostgreSQL
* CoffeeLot
* SensorMeasurement
* Alert
* routes lots :

  * POST /api/lots
  * GET /api/lots
  * GET /api/lots/fifo

Objectif de l’étape 3 :
Ajouter la gestion des mesures température/humidité et les alertes simples.

Important :
Ne pas ajouter MQTT pour l’instant.
Ne pas ajouter le backend central.
Ne pas ajouter le frontend.
On fait seulement les mesures et les alertes REST.

À créer :

1. Module Measurements :

* measurements.module.ts
* measurements.service.ts
* measurements.controller.ts
* dto/create-measurement.dto.ts

Routes attendues :

* POST /api/measurements
* GET /api/measurements
* GET /api/measurements/warehouse/:warehouseName

Le body attendu pour POST /api/measurements :

{
"warehouseName": "Warehouse Brazil 1",
"temperature": 34,
"humidity": 60
}

Quand une mesure est créée, elle doit être enregistrée en base PostgreSQL.

2. Logique d’alerte automatique :
   Pour le Brésil :

* température idéale : 29°C
* tolérance température : ±3°C
* donc plage acceptable : 26°C à 32°C
* humidité idéale : 55%
* tolérance humidité : ±2%
* donc plage acceptable : 53% à 57%

Si la température est hors plage, créer une alerte de type TEMPERATURE.
Si l’humidité est hors plage, créer une alerte de type HUMIDITY.

3. Module Alerts :

* alerts.module.ts
* alerts.service.ts
* alerts.controller.ts

Routes attendues :

* GET /api/alerts
* POST /api/alerts/check-expired-lots

La route POST /api/alerts/check-expired-lots doit vérifier les lots dont storageDate dépasse 365 jours.
Si un lot dépasse 365 jours :

* mettre son status à PERIME
* créer une alerte de type EXPIRED_LOT

4. Simulation email :
   Quand une alerte est créée, ne pas envoyer de vrai email pour l’instant.
   Afficher seulement un log dans la console :

EMAIL SIMULÉ ENVOYÉ À [brazil.manager@futurekawa.com](mailto:brazil.manager@futurekawa.com)
Type alerte : ...
Message : ...

5. Tests manuels à fournir :
   Donne-moi les commandes curl pour tester :

* création d’une mesure normale sans alerte
* création d’une mesure avec température trop haute
* création d’une mesure avec humidité trop haute
* consultation des alertes
* vérification des lots périmés

6. Explique brièvement le rôle de chaque nouveau fichier créé.

Ne touche pas à MQTT pour l’instant. MQTT sera l’étape 4.
